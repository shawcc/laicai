# 接入执行步骤

## 1. GitHub
1. 在本地初始化 Git：`git init`。
2. 添加远程仓库：`git remote add origin https://github.com/shawcc/laicai.git`。
3. 提交代码：`git add . && git commit -m "feat: launch AI world cup prediction site"`。
4. 推送主分支：`git branch -M main && git push -u origin main`。
5. 后续每个功能用 PR，GitHub Actions 会运行 `npm run check` 和 `npm run build`。

## 2. Vercel
1. 登录 Vercel，选择 `Add New Project`。
2. 导入 `shawcc/laicai` 仓库。
3. Framework 选择 `Vite`。
4. Build Command 使用 `npm run build`。
5. Output Directory 使用 `dist`。
6. 添加环境变量：`VITE_SUPABASE_URL`、`VITE_SUPABASE_ANON_KEY`。
7. 每个 PR 会自动生成 Preview URL，合并 `main` 后发 Production。

## 3. Supabase
1. 创建两个项目：`laicai-dev` 和 `laicai-prod`。
2. 在 SQL Editor 执行 `supabase/schema.sql`。
3. 在 Auth 里开启 Email Magic Link 和需要的 OAuth Provider。
4. 在 Project Settings 复制 `Project URL` 和 `anon public key`。
5. 写入 Vercel 环境变量和本地 `.env.local`。
6. 后续把当前 Mock 数据迁移到 `questions`、`question_options`、`ai_players`。

## 4. 大模型
1. 不要在前端保存模型 Key。
2. 用 Supabase Edge Function 或 Vercel Serverless Function 做 AI Gateway。
3. 公共 AI 情报员先生成 `evidence_packages`。
4. AI 选手答题时读取题目、选项和 `evidence_packages`。
5. 要求模型返回严格 JSON：`optionId`、`confidence`、`reasoning`、`usedEvidenceIds`。
6. 后端校验选项合法性和锁票状态后写入 `predictions`。

## 5. 动态发版
1. 内容更新走 Supabase 数据，不重新发版。
2. 功能更新走 Feature Flag，先部署但默认关闭。
3. PR Preview 验证通过后合并到 `main`。
4. 出问题优先关闭 Feature Flag，其次 Vercel 回滚 Deployment。
